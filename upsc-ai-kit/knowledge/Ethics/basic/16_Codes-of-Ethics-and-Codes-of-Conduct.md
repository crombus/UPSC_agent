**Core proposition:** ✅ ARC 2.2.6 — a Code of Ethics covers "broad guiding principles," while a
Code of Conduct must "in a precise and unambiguous manner, stipulate a list of acceptable and
unacceptable behaviour and action" — the two are complementary, not interchangeable.

## 2. Essential definitions

| Concept | Exam-ready meaning |
|---|---|
| ✅ **Code of Ethics** | Aspirational, principle-based standard (e.g., the seven Nolan principles) guiding *how* an official should aspire to act, without necessarily specifying sanctions. |
| ✅ **Code of Conduct** | Precise, rule-based, sanctionable list of specific dos and don'ts (e.g., gift limits, business-interest restrictions, disclosure deadlines). |
| ✅ **Ministers' Code of Conduct (India)** | Existing Government of India code (ARC 2.4.2) covering asset/liability disclosure, divestment from business interests, restrictions on gifts (symbolic gifts retained; non-symbolic gifts above ₹5,000 referred to the Toshakhana), and rules on foreign employment of family members. |
| ✅ **Rajya Sabha/Lok Sabha Committees on Ethics** | Parliamentary bodies overseeing legislators' "moral and ethical conduct," maintaining a Register of Members' Interests and recommending sanctions for breaches (ARC 2.5.2-2.5.3). |

## 3. Mechanism: how India's dual-code architecture is meant to work

1. **Code of Ethics** should be based on the Nolan-type seven principles (see `09`) and cover the
   Minister-civil servant relationship in broad terms (ARC 2.4.3).
2. **Code of Conduct** operationalises the aspirational principles into specific, checkable rules —
   e.g., ARC 2.4.2's Ministers' Code requires disclosure of assets/liabilities to the PM/CM, divestment
   from businesses supplying goods/services to government, and restrictions on accepting valuable
   gifts.
3. ✅ ARC 2.4.5's recommendations: create a Code of Ethics *in addition to* the existing Code of
   Conduct; set up dedicated monitoring units in the PM's/CM's offices; require an annual public
   report on violations; extend to coalition-government Ministers uniformly.
4. **Legislators** have their own parallel structure: Rajya Sabha's Code of Conduct (14 principles,
   ARC 2.5.2.2) and the Lok Sabha Ethics Committee's six general principles (ARC 2.5.3.1) both
   emphasise subordinating private interest to public trust, non-acceptance of fees/gifts for
   parliamentary acts, and maintaining "high standards of morality, dignity, decency."
5. ✅ 2025 GS-IV Q5(a) confirms this dual-code gap persists in the current exam's own framing: Code
   of Conduct "already in operation," Code of Ethics "not yet put in place" — asking candidates to
   "suggest a suitable model for code of ethics."

## 4. Indian applications and examples

- ✅ ARC 2.4.2's gift rule: symbolic gifts (sword of honour, ceremonial robes) may be retained;
  non-symbolic gifts valued over ₹5,000 must be assessed by the Toshakhana, with the Minister able
  to purchase the item by paying the difference above ₹5,000.
- ✅ ARC 2.5.2.2's Rajya Sabha Code of Conduct requires Members to resolve any conflict between
  personal interest and public trust by subordinating the former, and forbids accepting fees for
  votes, bills, resolutions or questions.
- ✅ The Mudgal case (1951) and the "cash-for-questions" case (2005) — both examined in `23` —
  illustrate the Code of Conduct's *enforcement* dimension (expulsion for proven violation), distinct
  from the aspirational Code of Ethics.

## 5. Must-Know Facts for Prelims

- ✅ ARC 2.2.6: Code of Ethics = broad principles; Code of Conduct = precise, sanctionable rules.
- ✅ The Rajya Sabha's Committee on Ethics was first constituted in 1997; the Lok Sabha's Committee
  on Ethics was constituted in 2001 (present committee reconstituted 2005 per ARC's account).
- ✅ ARC 2.4.2's Ministers' Code of Conduct fixes the Toshakhana gift-valuation threshold at ₹5,000.
- ⚠️ India, per the ARC's 2007 diagnosis and the 2025 GS-IV Q5(a) framing, still lacks a
  comprehensive Code of Ethics for Ministers, even though a Code of Conduct exists.

## 6. UPSC traps

- ❌ Code of Ethics and Code of Conduct are simply two names for the same document. -> ARC 2.2.6
  explicitly distinguishes them by function: aspirational principle vs precise, sanctionable rule.
- ❌ India has neither a Code of Conduct nor a Code of Ethics for Ministers. -> A Code of Conduct
  for Ministers already exists (ARC 2.4.2); it is the Code of Ethics that remains largely
  unimplemented, per both the ARC (2007) and the 2025 GS-IV paper's own framing.
- ❌ Parliamentary Ethics Committees can unilaterally punish a Member without House approval. -> The
  Committees recommend sanctions (e.g., expulsion); the House itself must accept/adopt the
  recommendation for it to take effect (ARC 2.5.7.2-2.5.7.3).

## 7. PYQ application

- ✅ 2025 GS-IV Q5(a): "Code of Conduct... already in operation... Code of Ethics is not yet put in
  place. Suggest a suitable model for code of ethics to maintain integrity, probity and transparency
  in governance" — the single most directly source-matching PYQ for this topic.
- ⚠️ Any question on parliamentary conduct/discipline should reference the Rajya Sabha/Lok Sabha
  Ethics Committees and cite a concrete precedent (Mudgal, cash-for-questions — see `23`).

## 8. Mains angles

- ⚠️ For 2025 GS-IV Q5(a)-type questions, propose a Code of Ethics model explicitly built on the
  seven Nolan principles (`09`), covering the Minister-civil-servant relationship, with a dedicated
  monitoring unit and annual public reporting — precisely the ARC's own 2.4.5 recommendation,
  updated for current institutional context.
- ⚠️ Always state clearly which existing Indian Code of Conduct provisions already exist, before
  proposing new Code of Ethics elements — this shows awareness of the current implementation gap
  rather than reinventing an already-existing wheel.

> **Answer thesis:** India's Code of Conduct architecture (Ministers, legislators, civil servants)
> is comparatively mature and enforceable, but its complementary Code of Ethics — the aspirational,
> Nolan-principle-based standard the ARC recommended in 2007 — remains substantially unimplemented,
> a persisting and currently examined gap.
