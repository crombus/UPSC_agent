## How this folder works

- **Two tiers per topic:** `basic/NN` builds the must-do skill — what the
  move is, why it matters, a starter version, and how to avoid the
  obvious failure. `advanced/NN` does not repeat basic — it adds
  taxonomy, tension pairs, levels/scale, stakeholder and causal analysis,
  paradox/dialectic testing, and a harder revision protocol. Read basic
  before advanced for any topic you have not studied before.
- **Tagging:** ✅ = an audited paper fact, a directly quoted prompt, or a
  claim tied to a dated official/primary source; ⚠️ = a pedagogical
  strategy, an analytical inference, or an internal heuristic — never an
  official UPSC rule; 📰 = a dated anchor (a specific, sourced, time-bound
  fact used only where genuinely time-sensitive); ❌ = a trap, a common
  misreading, or an explicit boundary of what this folder does not do.
- **Essay is a synthesis and expression layer**, not a duplicate GS
  content bank. It teaches candidates to *decode, argue, structure, and
  revise* — the substantive facts and mechanisms it illustrates with are
  owned by, and cross-linked to, the relevant GS subject module (see
  "Boundaries" below). No topic in this folder re-derives a full
  GS theory; each illustration ends with a `Source module:` pointer.
- **No invented official rubric.** Neither audited paper specifies a
  granular UPSC marking scheme, an examiner weightage table, a "topper
  formula," or an ideal paragraph/example count. Topic `15`'s scoring
  grid is explicitly and repeatedly labelled **"Internal Essay Practice
  Rubric — not UPSC's official rubric."** Nothing in this folder should
  ever be read as UPSC's own marking key.
- **No full model essays.** Every topic gives decode grids, thesis
  alternatives, argument maps, paragraph plans, transition banks, error
  logs and timed drills — never a complete finished essay to memorise and
  reproduce, because that defeats the exam's test of original synthesis.

### ❌ Source-paper defects preserved, not corrected

Both papers contain printed defects. This folder quotes each prompt and
instruction **exactly as printed**, then labels the defect — it never
silently "fixes" the source:

- **2024 instruction, exact:** "Write two essays, choosing one topic from
  each of the following Sections A and B, in about 1000-1200 words
  each:" followed by "(125 × 2 = 250)." Section B **restarts its own
  numbering at 1–4** rather than continuing 5–8.
- **2024 A2, exact:** "The empires of the **futures** will be the empires
  of the mind." — prints "futures," not "future."
- **2024 A3, exact:** "There is no path to happiness, **Happiness** is
  the  path." — mid-sentence capital "Happiness" and a printed double
  space before "path."
- Neither 2024 nor 2025 prompts carry any **author attribution** in the
  source paper.
- **2025 instruction, exact, including its own defect:** "Write two
  essays, choosing one topic from each of the following **Sections A
  as** in about 1000 – 1200 words each:" — ordinary grammar would expect
  "Sections A and B"; the paper as printed reads "Sections A as." This
  folder quotes it verbatim and explains the operative requirement
  (choose one topic from each displayed Section) rather than silently
  rewriting the sentence.

## Boundaries with other subject modules

Essay does not own or duplicate any GS subject's factual/theoretical
core. It borrows a mechanism, a case, or a value only long enough to
serve an argument, then points back to the owning module:

| Subject module | What Essay borrows | What Essay does not do |
|---|---|---|
| `Indian-Society` | social mechanisms, diversity, family/community cases | full sociological theory notes |
| `International-Relations` | diplomacy, conflict and global-order examples | bilateral chronology and treaty-by-treaty detail |
| `Economy` | growth, inequality, work and consumption mechanisms | macroeconomic definitions and data tables |
| `Ethics` | values, dilemmas and thinkers used as analytical lenses | full thinker biographies or case-study machinery |
| `Environment-and-Ecology` | ecological causality and sustainability examples | ecology, law and convention-by-convention coverage |
| `Science-and-Technology` | innovation, scientific temper and platform/AI cases | technical architecture detail |
| `Modern-Indian-History` / `Ancient-Indian-History` / `Medieval-Indian-History` | historical analogy and civilisational perspective | chronological factual notes |
| `Indian-Art-and-Culture` | pluralism, meaning and lived-tradition illustrations | art-form/reference catalogues |
| `Governance` / `Polity` | institutions, state capacity and policy trade-offs | scheme-by-scheme administrative detail or constitutional-article machinery |
| `Internal-Security` | conflict/deterrence framing used as an IR-adjacent lens | operational security-doctrine detail |

A reusable Essay illustration always carries the chain **claim → one
verified fact/example → essay function → limitation → source-module
link** (topic `12`), never a bare fact dump borrowed from another folder.

## Navigation

1. Read `00_Master-Framework.md` first — it is the decode → expand →
   thesis → argument → evidence → structure → conclusion → revision
   answer engine every topic below plugs into, plus the summary routing
   table for all 16 audited prompts.
2. New to essay writing: read `basic/01` through `basic/09` in order,
   then `basic/13`–`16` for execution and practice.
3. Already comfortable with structure: go straight to the matching
   `advanced/NN` file for the skill you are weakest in, using its
   `Companion:` link back to the basic file only if a foundational gap
   appears.
4. Before any timed attempt, read `advanced/16` (or `basic/16`) for the
   routing/outline of the specific prompt you intend to practise.
5. After every attempt, log it using topic `15`'s internal rubric and
   topic `16`'s error log — never skip the revision-loop step.
