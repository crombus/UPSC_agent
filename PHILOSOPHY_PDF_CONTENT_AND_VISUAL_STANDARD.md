# Philosophy Optional PDF — Content and Visual Standard

This is the mandatory standard for every topic-wise UPSC Philosophy Optional PDF.
It complements `PHILOSOPHY_VISUAL_NOTES_AI_PLAYBOOK.md`.

## 1. Governing Principle

Content determines the design. Never reduce doctrine, criticism, comparisons,
PYQs or answer-writing material merely to create a shorter or prettier PDF.

The target is a complete, revision-ready register:

- academically substantive;
- visually understandable;
- directly grounded in local Philosophy books;
- mapped to the official syllabus and PYQs;
- usable for 10-, 15- and 20-mark answers.

## 2. Mandatory Pre-Build Workflow

1. Read the exact UPSC syllabus line.
2. Analyze the relevant local book PDFs directly.
3. Cross-check with the verified Philosophy knowledge file.
4. Read every available topic PYQ.
5. Research current educational information-design ideas.
6. Identify the best visual for each learning problem.
7. Build the complete content outline before designing pages.
8. Generate the PDF only after content coverage is complete.

Do not replace direct local-PDF analysis with Qdrant summaries.

## 3. Mandatory Content Depth

Every topic must include, where relevant:

1. Historical origin and intellectual context.
2. Definitions and technical vocabulary.
3. Complete exposition of every syllabus sub-part.
4. Internal argument or derivation.
5. Major criticisms and replies.
6. Intra-school and inter-thinker comparisons.
7. Indian or Western cross-links where philosophically relevant.
8. Must-know facts.
9. UPSC traps and common misreadings.
10. Verified PYQs and recurring question patterns.
11. Applied-question drills.
12. Answer frameworks for 10, 15 and 20 marks.
13. A qualified philosophical judgement.
14. Direct source-page or source-chapter references.

The Plato–Aristotle reference sample contains approximately 3,500 words across
13 pages. Topic length may vary, but completeness must not be sacrificed.

## 4. Visual Selection Rules

Use the visual that best explains the reasoning task:

| Learning problem | Preferred visual |
|---|---|
| Whole doctrine | Concept map or conceptual landscape |
| Claim–objection–reply | Argument tree |
| Causal or logical sequence | Flowchart |
| Multi-axis comparison | Comparison matrix |
| Limited overlap | Venn comparison lens |
| Historical development | Layered timeline |
| School influence | Relationship or metro map |
| Technical structure | Labelled diagram |
| Abstract doctrine | Topic-specific visual metaphor |
| Answer writing | Answer spine or dialectical scaffold |

Venn diagrams are examples, not defaults. Visuals must improve comprehension.

## 5. Pagination and Boundary Rules

- Never split tables, facts panels, UPSC-traps panels, memory hooks, diagrams,
  Mains-angle panels or Study-link panels across pages.
- If a complete block does not fit, begin it on the next page.
- Never isolate a heading at the bottom of a page.
- Keep all diagram text inside measured nodes or bounded detail cards.
- Use useful opening-page whitespace for a topic-specific concept visual.
- Never fill space with unrelated decoration.

## 6. Required Quality Audit

Before completion:

1. Confirm every syllabus sub-part is present.
2. Confirm all relevant PYQs are represented.
3. Confirm no page is empty.
4. Confirm no text or diagram crosses page boundaries.
5. Confirm no semantic block is split.
6. Confirm Unicode and transliteration render correctly.
7. Render and inspect representative pages.
8. Delete temporary `_data.py` files only after successful generation.
